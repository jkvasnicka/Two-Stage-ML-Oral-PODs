Preliminary workflow for supervised machine learning
regression & classification tasks

Date: March 21, 2023

Prof. Chiu and I are evaluating the performances of supervised ML models at predicting toxicity outcomes of chemicals (N = ~1,000), for both a continuous target variable and several binary targets (positive or negative for different organ systems). We are thinking of trying different estimators, short-listing a few top-performing models, and then perhaps building an ensemble of top-performers for the final models.

I had a first pass at adapting the general workflow Prof. Wright recommended over email exchanges, as shown below. I would appreciate any feedback on this implementation. In addition, I have a some follow-up questions toward the end of this notebook.

Preliminary workflow: continuous target (regression task)¶

Previously, I constructed a random forest with a single train-test split. Following our email exchanges, I tried to improve this model by adapting Prof. Wright's suggestions for incorporating cross validation. I also incorporated preliminary hyperparameter tuning using a nested cross-validation strategy and a hyperparameter grid search. I read that using a nested strategy is generally best practice when implementing hyperparameter tuning, in order to avoid overfitting. Let's walk through the implementation.

# Initialize container. 
cv_for_repeat = {}

# For each repeat, use a different random state for the nested CV splitters.
for i in range(n_repeats):
    print(f'Repeat {i}')

    # Configure the outer cross-validation procedure.
    # Returns a list of randomized indices in which to split.
    cv_outer = KFold(n_splits=n_splits_outer, shuffle=True, random_state=i)

    # Initialize container. 
    results_for_fold = []
    
    # Implement the nested cross validation.
    for j, (train_ix, test_ix) in enumerate(cv_outer.split(X)):
        print(f'  Outer loop {j}...')

        # Split the data. 
        X_train, X_test = X[train_ix, :], X[test_ix, :]
        y_train, y_test = y[train_ix], y[test_ix]

        # Configure the inner cross-validation procedure for the grid search.
        cv_inner = KFold(n_splits=n_splits_inner, shuffle=True, random_state=i)

        # Initialize the hyperparameter search grid. By default, the search 
        # learns the parameters that maximize the mean score (R2) qcross the 
        # inner CV loop.
        search = GridSearchCV(
            estimator=pipe, 
            param_grid=param_grid, 
            scoring='r2', 
            cv=cv_inner,
            refit=True,
            n_jobs=-1)

        # Implement the grid search. 
        result = search.fit(X_train, y_train)

        # Find the "best" model fit on the full training set.
        best_model = result.best_estimator_

        # Evaluate the model on the held-out test set. 
        yhat = best_model.predict(X_test)
        
        results_for_fold.append( 
            {'best_mean_score': result.best_score_, 
            **result.best_params_})
    
    cv_for_repeat[i] = pd.DataFrame(results_for_fold)
    
cv_for_repeat = pd.concat(cv_for_repeat)
cv_for_repeat.index.names = ['repeat', 'outer_fold']

Potential next steps¶

1. More extensive feature preprocessing¶

  1. Drop constant features
  2. Drop features with missing values exceeding threshold % (10 default)
  3. Impute missing values with medians
  4. Apply log10 and/or other transformations to reduce skewness
  5. Center & scale continuous features
  6. Remove strong correlations among features (default > 0.9, Pearson)
